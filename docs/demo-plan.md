# Demo Plan

## Objective
The objective of this demo is to present the UniEvent system, demonstrate its core functionalities, and showcase the development process including GitHub repository, CI/CD pipeline, and project management approach.

---

## Demo Flow

### 1. Project Introduction
- Brief overview of UniEvent
- Purpose of the platform (event creation and participation)

---

### 2. System Demonstration (MVP)
- Create a new event
- View existing events
- Register for an event (if available)

---

### 3. GitHub Repository
- Show project structure (backend, frontend, docs)
- Explain documentation:
  - Risk Analysis
  - Validation & Testing Plan
  - Risk Updates
  - Demo Plan

---

### 4. CI/CD Pipeline
- Show CI/CD configuration (GitHub Actions / Docker)
- Briefly explain automated build and deployment process

---

### 5. Project Management (Sprint Board)
- Show project board (Jira / GitHub Projects)
- Explain task distribution and sprint workflow

---

## Expected Outcome
At the end of the demo:
- The system should run successfully
- Core features should work correctly
- Development process should be clearly demonstrated
